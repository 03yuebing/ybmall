# 01. 项目初始化过程总结

这份笔记总结我们从零创建 `ybmall` 项目的过程。重点不是背配置，而是理解：为什么要这么建、每个 `pom.xml` 管什么、为什么 `ybmall-product` 能启动。

## 1. 我们现在在做什么

原始 `gulimall-learning` 是一个 2020 年左右的谷粒商城学习项目，技术栈比较旧。我们现在采用的方式是：

- 原仓库：作为参考答案和业务流程教材。
- `ybmall`：自己手动搭建的现代版练习项目。

这样做的目的不是复制代码，而是通过手动搭建理解一个微服务电商项目的骨架。

## 2. 当前项目结构

目前项目结构是：

```text
ybmall
├── pom.xml
├── ybmall-common
│   ├── pom.xml
│   └── src
└── ybmall-product
    ├── pom.xml
    └── src
```

各部分作用：

- `ybmall`：父工程，只负责统一管理子模块、版本、依赖和插件。
- `ybmall-common`：公共模块，后面放统一返回对象、异常、常量、工具类等。
- `ybmall-product`：商品服务，目前已经可以作为一个 Spring Boot Web 服务启动。

父工程下面不保留 `src` 目录，因为父工程不写业务代码。

## 3. 为什么要用父工程

一个真实的商城会有很多服务：

```text
商品服务、订单服务、库存服务、会员服务、购物车服务、搜索服务、认证服务……
```

如果每个服务都自己管理 JDK 版本、Spring Boot 版本、依赖版本，后面会很乱。

所以我们先创建一个父工程 `ybmall`，让它统一管理：

- 项目坐标：`groupId`、`artifactId`、`version`
- JDK 版本
- Spring Boot 版本
- 子模块列表
- 后续公共依赖版本
- 后续 Maven 插件配置

可以把父工程理解为“总公司”，各个子模块是“部门”。

## 4. 父工程 pom.xml 做了什么

父工程 `pom.xml` 的核心内容：

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.5.0</version>
    <relativePath/>
</parent>

<groupId>com.yuebing.ybmall</groupId>
<artifactId>ybmall</artifactId>
<version>0.0.1-SNAPSHOT</version>
<packaging>pom</packaging>
```

这里最重要的是两层父子关系。

第一层：

```text
ybmall 继承 spring-boot-starter-parent
```

作用是让 Spring Boot 帮我们管理大量依赖和插件版本。比如我们写：

```xml
<artifactId>spring-boot-starter-web</artifactId>
```

通常不用再手写版本号，因为 Spring Boot 父工程已经帮我们管理好了。

第二层：

```text
ybmall-common 和 ybmall-product 继承 ybmall
```

作用是让子模块继承我们自己项目的统一配置。

## 5. packaging=pom 是什么意思

父工程里有：

```xml
<packaging>pom</packaging>
```

它表示：这个模块不是普通 jar 包，也不是可启动服务，而是一个聚合管理工程。

也就是说：

```text
ybmall 本身不启动
ybmall-product 才启动
```

这是 Maven 多模块项目里很关键的概念。

## 6. modules 是什么意思

父工程里有：

```xml
<modules>
    <module>ybmall-common</module>
    <module>ybmall-product</module>
</modules>
```

它告诉 Maven：

```text
这个父工程下面有两个子模块：
1. ybmall-common
2. ybmall-product
```

以后在根目录执行 Maven 构建时，Maven 会知道要一起处理这些模块。

## 7. properties 是什么意思

父工程里配置了：

```xml
<properties>
    <java.version>17</java.version>
    <maven.compiler.release>${java.version}</maven.compiler.release>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
</properties>
```

含义：

- `java.version=17`：整个项目统一使用 JDK 17。
- `maven.compiler.release=17`：编译出来的代码按 Java 17 标准处理。
- `project.build.sourceEncoding=UTF-8`：源码统一使用 UTF-8 编码。

这里的重点是：JDK 版本由父工程统一管理，子模块不要重复写一份。

## 8. common 模块现在做什么

`ybmall-common` 当前还没有写业务代码，但它已经是一个正式 Maven 子模块。

它的 `pom.xml` 继承父工程：

```xml
<parent>
    <groupId>com.yuebing.ybmall</groupId>
    <artifactId>ybmall</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</parent>

<artifactId>ybmall-common</artifactId>
```

以后这里会放：

- 统一响应结果类，例如 `R`
- 统一异常类
- 常量
- 工具类
- 通用校验分组
- 跨模块共享的 DTO / VO

现在先空着是正常的。

## 9. product 模块做了什么

`ybmall-product` 是我们的第一个业务服务。

它的 `pom.xml` 里做了三件事。

第一，继承父工程：

```xml
<parent>
    <groupId>com.yuebing.ybmall</groupId>
    <artifactId>ybmall</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</parent>
```

第二，声明自己的模块名：

```xml
<artifactId>ybmall-product</artifactId>
```

第三，引入依赖：

```xml
<dependencies>
    <dependency>
        <groupId>com.yuebing.ybmall</groupId>
        <artifactId>ybmall-common</artifactId>
        <version>${project.version}</version>
    </dependency>

    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>
```

这三个依赖分别表示：

- `ybmall-common`：商品服务可以使用公共模块里的类。
- `spring-boot-starter-web`：商品服务具备 Web 服务能力，内置 Tomcat，可以接收 HTTP 请求。
- `spring-boot-starter-test`：后续写测试使用。

## 10. 这次 Maven 报错学到了什么

之前 `ybmall-product` 曾经错误地写成依赖：

```xml
<groupId>com.yuebing</groupId>
<artifactId>ybmall</artifactId>
```

Maven 报错：

```text
Could not find artifact com.yuebing:ybmall:jar:0.0.1-SNAPSHOT
```

原因是：`ybmall` 是父工程，`packaging=pom`，它不是一个业务 jar 包。

正确关系应该是：

```text
ybmall-product 的 parent 是 ybmall
ybmall-product 的 dependency 是 ybmall-common
```

也就是：

```text
parent：继承配置
dependency：使用代码
```

这是 Maven 多模块项目里非常重要的一条线。

## 11. 启动类做了什么

我们在 `ybmall-product` 中创建了启动类：

```text
src/main/java/com/yuebing/ybmall/product/YbmallProductApplication.java
```

核心代码：

```java
@SpringBootApplication
public class YbmallProductApplication {
    public static void main(String[] args) {
        SpringApplication.run(YbmallProductApplication.class, args);
    }
}
```

`@SpringBootApplication` 表示这是一个 Spring Boot 应用入口。

它背后主要包含三层意思：

1. 开启自动配置。
2. 扫描当前包和子包里的 Spring 组件。
3. 声明这是一个配置类。

所以启动类放在：

```text
com.yuebing.ybmall.product
```

后续 Controller、Service、Mapper 建议放在它的子包下：

```text
com.yuebing.ybmall.product.controller
com.yuebing.ybmall.product.service
com.yuebing.ybmall.product.dao
```

这样 Spring Boot 才能自动扫描到。

## 12. application.yml 做了什么

`ybmall-product` 里有配置文件：

```text
src/main/resources/application.yml
```

当前核心配置：

```yaml
server:
  port: 10001

spring:
  application:
    name: ybmall-product
```

含义：

- `server.port=10001`：商品服务启动在 10001 端口。
- `spring.application.name=ybmall-product`：这个服务的名字叫 `ybmall-product`。

现在还没有接 Nacos，所以服务名暂时只是本地标识。后面接入注册中心后，这个名字会被 Nacos 用来识别服务。

注意：如果你的注释里还写着“10000 端口”，记得手动改成“10001 端口”，避免以后误解。

## 13. 为什么启动成功很重要

启动日志中出现：

```text
Tomcat initialized with port 10001
Tomcat started on port 10001
Started YbmallProductApplication
```

说明：

- JDK 17 正常。
- Maven 依赖正常。
- Spring Boot 正常。
- 内置 Tomcat 正常。
- `ybmall-product` 已经是一个可运行的 Web 服务。

这是我们从零搭建项目的第一个里程碑。

## 14. 到目前为止的完整过程

我们已经完成：

1. 用 IDEA 创建父工程 `ybmall`。
2. 创建两个子模块 `ybmall-common` 和 `ybmall-product`。
3. 删除父工程多余的 `src` 目录。
4. 将项目 JDK 调整为 17。
5. 将父工程改成 `packaging=pom`。
6. 在父工程中注册两个子模块。
7. 让两个子模块继承父工程。
8. 在父工程中引入 Spring Boot 父工程。
9. 在 `ybmall-product` 中加入 Web 依赖。
10. 修正 `ybmall-product` 对 `ybmall-common` 的依赖。
11. 创建 `YbmallProductApplication` 启动类。
12. 创建 `application.yml`。
13. 将商品服务端口设置为 `10001`。
14. 启动商品服务成功。

## 15. 当前你需要真正记住的概念

先记住这几组关系：

```text
父工程 ybmall
  -> 管理 common 和 product
```

```text
ybmall-product
  -> parent 继承 ybmall
  -> dependency 依赖 ybmall-common
```

```text
Spring Boot starter parent
  -> 管理 Spring Boot 依赖版本
```

```text
spring-boot-starter-web
  -> 让 product 成为 Web 服务
```

```text
application.yml
  -> 管理当前服务的运行配置
```
