# 02. 模块架构说明

这份文档记录 `ybmall` 当前的模块划分。现在这些模块大多数还只是空 Maven 子模块，先写清楚边界，是为了后面开发时知道代码应该放在哪里。

## 1. 当前模块总览

```text
ybmall
├── ybmall-common
├── ybmall-product
├── ybmall-member
├── ybmall-coupon
├── ybmall-ware
├── ybmall-order
├── ybmall-gateway
├── ybmall-third-party
├── ybmall-search
├── ybmall-auth-server
├── ybmall-cart
└── ybmall-seckill
```

这些模块对应一个电商系统的主要业务能力：

```text
商品、会员、优惠、库存、订单、网关、第三方、搜索、认证、购物车、秒杀
```

## 2. ybmall-common

公共模块，不是独立服务。

未来主要放：

- 统一响应结果，例如 `R`
- 统一异常类
- 错误码枚举
- 常量
- 工具类
- 通用校验分组
- 跨模块复用的 DTO / VO

它的特点是：

```text
被其他模块依赖
自己不启动
不直接处理 HTTP 请求
```

例如 `ybmall-product` 已经依赖了它。

## 3. ybmall-product

商品服务，是我们第一个启动成功的业务服务。

未来负责：

- 商品分类
- 品牌管理
- 属性分组
- 规格参数
- 销售属性
- SPU 管理
- SKU 管理
- 商品发布
- 商品详情
- 商品上下架

它在电商系统里非常核心，因为订单、购物车、搜索、秒杀都会用到商品数据。

后续学习时，优先从这里开始，因为商品服务能把数据库表、CRUD、业务查询、缓存、服务调用串起来。

## 4. ybmall-member

会员服务。

未来负责：

- 用户信息
- 会员等级
- 收货地址
- 会员积分
- 会员成长值
- 登录记录
- 收藏记录

它主要回答这些问题：

```text
这个用户是谁？
他的收货地址是什么？
他的会员等级和积分是多少？
```

订单服务在结算时会调用会员服务查询收货地址。

## 5. ybmall-coupon

优惠服务。

未来负责：

- 优惠券
- 满减
- 折扣
- 会员价格
- 商品积分
- 秒杀活动场次
- 首页营销活动

它主要回答：

```text
这个商品能不能优惠？
这个用户有没有优惠券？
这个活动什么时候开始？
```

商品发布时可能会保存积分、满减、会员价等营销信息。

## 6. ybmall-ware

库存服务，也可以理解为仓储服务。

未来负责：

- 仓库信息
- 商品库存
- 采购需求
- 采购单
- 锁库存
- 解锁库存

它主要回答：

```text
这个商品有没有库存？
下单时能不能锁住库存？
订单取消后库存怎么释放？
```

订单提交时会调用库存服务锁库存。

## 7. ybmall-order

订单服务。

未来负责：

- 订单确认页
- 提交订单
- 订单验价
- 防重复提交
- 订单创建
- 订单列表
- 订单详情
- 支付回调
- 订单关闭

它是业务流程最复杂的模块之一。

典型流程：

```text
购物车选中商品
  -> 进入结算页
  -> 查询收货地址
  -> 查询商品价格
  -> 提交订单
  -> 锁库存
  -> 创建订单
  -> 支付
```

后面会结合 RabbitMQ 学订单超时关闭和库存自动解锁。

## 8. ybmall-gateway

网关服务，是所有外部请求进入微服务系统的统一入口。

未来负责：

- 请求路由
- 路径转发
- 跨域配置
- 登录校验入口
- 限流入口
- 灰度或统一过滤逻辑

它主要回答：

```text
这个请求应该交给哪个服务处理？
```

例如：

```text
/api/product/** -> ybmall-product
/api/member/**  -> ybmall-member
/api/order/**   -> ybmall-order
```

网关本身通常不写具体业务逻辑。

## 9. ybmall-third-party

第三方服务。

未来负责对接外部平台：

- 对象存储 OSS
- 短信服务
- 邮件服务
- 第三方 API

它的作用是把外部服务包装成内部统一接口，避免每个业务服务都直接操作外部平台。

例如注册时发送短信验证码，可以由认证服务调用第三方服务完成。

## 10. ybmall-search

搜索服务。

未来负责：

- 商品上架到 Elasticsearch
- 商品全文检索
- 分类筛选
- 品牌筛选
- 属性筛选
- 排序
- 分页
- 聚合统计

它主要回答：

```text
用户搜索关键词后，应该展示哪些商品？
```

搜索服务不会直接承担商品管理，它更像是商品数据的搜索视图。

## 11. ybmall-auth-server

认证服务。

未来负责：

- 登录页
- 注册页
- 短信验证码
- 用户登录
- 用户注册
- 第三方登录
- 分布式 Session

它主要回答：

```text
用户是谁？
用户有没有登录？
登录状态如何在多个服务之间共享？
```

后面会结合 Redis 学分布式 Session。

## 12. ybmall-cart

购物车服务。

未来负责：

- 添加购物车
- 查询购物车
- 修改商品数量
- 勾选商品
- 删除商品
- 未登录购物车
- 登录购物车
- 登录后合并购物车

它主要回答：

```text
用户想买哪些商品？
哪些商品被勾选准备下单？
```

购物车数据后面会放到 Redis 中。

## 13. ybmall-seckill

秒杀服务。

未来负责：

- 秒杀活动商品上架
- 秒杀商品缓存预热
- 秒杀随机码
- 抢购校验
- 防重复抢购
- 库存快速扣减
- 异步创建秒杀订单

它主要解决高并发场景：

```text
很多用户同时抢少量商品，系统怎样不崩？
```

后面会结合 Redis、RabbitMQ、限流学习。

## 14. 模块之间的大致关系

可以先这样理解：

```text
浏览器 / 前端
  -> ybmall-gateway
  -> 具体业务服务
```

常见调用关系：

```text
ybmall-order
  -> ybmall-member
  -> ybmall-cart
  -> ybmall-product
  -> ybmall-ware
```

```text
ybmall-product
  -> ybmall-coupon
  -> ybmall-search
```

```text
ybmall-auth-server
  -> ybmall-member
  -> ybmall-third-party
```

这些调用现在还没有实现，先建立脑图即可。

## 15. 当前开发状态

目前已经完成：

- 父工程 `ybmall`
- 12 个业务/基础模块
- `ybmall-product` 已经加入 Spring Boot Web 依赖
- `ybmall-product` 已经创建启动类
- `ybmall-product` 已经能在 `10001` 端口启动

目前还没有完成：

- Controller
- 数据库连接
- MyBatis-Plus
- Nacos
- Gateway 路由
- Redis
- Elasticsearch
- RabbitMQ
- 具体业务表和业务代码

## 16. 后续建议学习顺序

建议按这个顺序继续：

1. `ybmall-product` 创建第一个 Controller。
2. `ybmall-product` 接入 MySQL 和 MyBatis-Plus。
3. 实现商品分类基础 CRUD。
4. 抽取统一返回对象到 `ybmall-common`。
5. 接入 Nacos。
6. 创建并启动 `ybmall-gateway`。
7. 通过网关访问商品服务。
8. 再逐步扩展会员、优惠、库存、订单等模块。

现在先不要急着让所有模块都能启动。真实开发里也是先让一个核心服务跑通，再扩展其他服务。
